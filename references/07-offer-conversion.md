# Offer & Conversion

Use this reference when content must create leads, bookings, inbox messages, trials, or purchases.

## Offer Components

```txt
1. Desired result
2. Target customer
3. Mechanism
4. Timeframe or path to progress
5. Proof
6. Bonus or added value
7. Risk reduction
8. Clear CTA
```

## Offer Formula

```txt
Giúp [đối tượng]
đạt [kết quả mong muốn]
mà không cần [rào cản lớn]
bằng [cơ chế khác biệt]
trong [khoảng thời gian hợp lý hoặc lộ trình rõ ràng].
```

If a specific timeframe is not proven, use:

```txt
theo một lộ trình rõ ràng
từng bước
với quy trình đã được chuẩn hóa
```

## CTA Types

```txt
Low friction:
Nhận checklist, xem demo, nhận audit, tải template, comment keyword.

Medium friction:
Đăng ký tư vấn, điền form, đặt lịch, nhắn tin để nhận đề xuất.

High intent:
Mua ngay, bắt đầu gói, đặt cọc, đăng ký triển khai.
```

## CTA Discipline

```txt
Use one primary CTA per content piece.
If a secondary link is necessary, explain its role outside the main copy.
Do not place two equal CTAs next to each other without priority.
Match CTA friction to journey stage and trust level.
```

CTA priority examples:

```txt
Primary: Xem demo skill.
Secondary: Xem roadmap phiên bản.

Primary: Đăng ký nhận audit.
Secondary: Đọc case study trước khi đăng ký.
```

## Conversion Rules

```txt
Match CTA to journey stage.
Unaware audiences need low-friction CTAs.
Product aware and most aware audiences can receive direct CTAs.
If trust is weak, add proof before CTA.
If offer is vague, clarify result, mechanism, and risk reducer.
If CTA is split across multiple links, assign a primary and secondary role.
```
