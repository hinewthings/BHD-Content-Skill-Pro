# Trust, Proof & Objection

Use this reference for any content that needs belief before action.

## Trust Layers

```txt
Familiarity Trust:
Consistent presence, recognizable voice, repeated topics, series, useful comments, community interaction.

Evidence Trust:
Reviews, testimonials, case studies, numbers, demos, before-after, screenshots, real customers, partners, press, certificates.

Behavior Trust:
Clear process, honest limitations, refund/warranty/support policy, public issue handling, realistic claims, customer-first behavior.
```

## Proof Types

```txt
Customer proof: testimonials, reviews, case studies.
Data proof: metrics, benchmarks, reports, before-after numbers.
Process proof: methodology, checklist, SOP, workflow, audit.
Demo proof: screen recording, product walkthrough, sample output.
Authority proof: expert credentials, press, certificates, partnerships.
Community proof: comments, user questions, repeated demand.
Logic proof: reasoned explanation when hard proof is not available.
```

For claims with numbers, legal language, customer outcomes, testimonials, or market-wide statements, grade proof using `19-proof-grading.md` before deciding claim strength.

## No-Proof Fallback

If proof is missing, do not invent it. Use:

```txt
- Process explanation
- Demo suggestion
- Checklist
- Sample audit
- Transparent assumption
- "Bạn có thể thay bằng số liệu thật ở đây"
```

```txt
Cần hỏi thêm người khác
Không biết có phù hợp không
Sợ bị lừa
Muốn tự làm
```

## Objection Handling Formula

```txt
Acknowledge concern
Explain why the concern is valid
Reframe the real risk or decision
Provide proof or process
Reduce risk
Suggest a small next step
```
