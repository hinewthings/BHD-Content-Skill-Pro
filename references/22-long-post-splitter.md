# Long Post Splitter

Use this reference when a post is long, dense, has too many ideas, or the user asks for repurposing/series/calendar.

The goal is to preserve strategic depth while improving readability and distribution.

## Split Triggers

Recommend splitting when:

```txt
The post is over 900-1200 Vietnamese words.
The post has more than 4 major ideas.
The hook promises one topic but body covers several topics.
The post mixes education, proof, product pitch, policy, and roadmap.
The CTA appears after too much cognitive load.
The content would work better as a nurture sequence or series.
```

Do not force splitting when:

```txt
The user asks for long-form thought leadership.
The platform is LinkedIn long-form, blog, newsletter, or landing page.
The audience is highly sophisticated and expects depth.
```

## Split Patterns

```txt
Problem series:
Post 1: The market/problem
Post 2: Why current solutions fail
Post 3: The better mechanism
Post 4: Proof/demo
Post 5: Offer/CTA

Education series:
Post 1: Core insight
Post 2: Framework
Post 3: Mistakes
Post 4: Examples
Post 5: Checklist

Product roadmap series:
Post 1: Why the product exists
Post 2: What v1 solves
Post 3: What v2/v3 improves
Post 4: What v4 unlocks
Post 5: Upgrade/registration CTA

Founder POV series:
Post 1: Personal observation
Post 2: What changed in the market
Post 3: What I built
Post 4: What I learned from users
Post 5: What to try next
```


## Compression Rule

If keeping as one post:

```txt
Keep one core thesis.
Limit to 3 sections.
Move proof/source notes to the end.
Use one CTA.
Cut repeated claim types.
Convert feature lists into benefit clusters.
```
